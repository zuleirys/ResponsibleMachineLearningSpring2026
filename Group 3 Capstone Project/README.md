# DNSC 6330: Responsible Machine Learning — Capstone Project
## Group 3 | 2024 HMDA LAR Mortgage Approval Classification

---

## Project Overview

This repository contains the full capstone analysis for DNSC 6330 (Responsible Machine Learning) at The George Washington University, School of Business. The project develops a binary classification model to predict mortgage loan approval (`action_taken`) using the 2024 Home Mortgage Disclosure Act (HMDA) Loan/Application Records (LAR) dataset, and applies the responsible ML principles taught in class across fairness, explainability, and robustness dimensions.

**Optimization objective:** Maximize F1-score subject to the EEOC 80% rule holding across racial groups.

**Capstone questions answered:**
1. What is the optimization objective?
2. What are the known failure modes?
3. How are subgroup errors measured?
4. What risks remain after mitigation?
5. Is deployment defensible?

### Label Creation

Per capstone specification:

| `action_taken` Value | Meaning | Label |
|---|---|---|
| 1 | Loan originated | **1 (approved)** |
| 2 | Approved but not accepted | **1 (approved)** |
| 3 | Application denied | **0 (denied)** |
| 4–8 | Withdrawn, incomplete, purchased, etc. | **Filtered out** |

**Note:** Filtering values 4–8 introduces sample selection bias — withdrawn and incomplete applications are non-random across protected groups and represent 29.4% of raw records.

## Notebooks

### NB01 — Exploratory Data Analysis

**Goals:** Understand distributions, class balance, missingness, and group base rates. Establish a pre-model fairness baseline.

**Key outputs:**
- Class distribution: 75.69% approval rate, significant imbalance
- Missing values: DTI (10.9%), property_value (sentinel codes); interest_rate removed due to leakage
- Pre-model 80% rule: Black, AI/AN, and Native Hawaiian/Pacific Islander applicants all fail the EEOC 80% rule in raw data
- Base rate spread: 38.4pp between highest (Joint: 80.4%) and lowest (Free Form Text: 42.1%) groups

### NB02 — Preprocessing

**Goals:** Clean, encode, split, impute — fully reproducible pipeline.

**Key decisions:**
- HMDA null encodings (`Exempt`, blank) replaced with `np.nan`
- Income sentinel values (>9,999 and <0) set to NaN
- Property value int32 sentinel (2,147,483,647) set to NaN
- Loan term values >480 months set to NaN
- DTI range strings parsed to numeric midpoints
- Credit score type columns collapsed to binary (`has_credit_score`, `co_applicant_has_credit_score`)
- `get_dummies` encoding for categoricals (`drop_first=True`)
- Protected columns held out of X entirely
- Stratified 80/20 split (`random_state=42`)
- Median imputation fitted on train only to prevent leakage

### NB03 — Modeling

**Goals:** Baseline model, primary model selection, threshold tuning.

**Models evaluated:**

| Model | Test AUC | Test F1 | F1 (Denial) | G_gap AUC |
|---|---|---|---|---|
| Logistic Regression | 0.7463 | 0.7866 | 0.5262 | -0.0013 |
| Random Forest | 0.8414 | 0.8532 | 0.6177 | -0.0025 |
| **Gradient Boosting** | **0.8515** | **0.9001** | **0.4979** | **-0.0019** |

**Key decisions:**
- Gradient Boosting selected: highest AUC (0.8515) and overall F1 (0.9001)
- Trained on 500k stratified subsample for computational tractability; evaluated on full 1.7M test set
- `RandomizedSearchCV` with 8 iterations × 5 folds on 100k subsample
- Best params: `subsample=0.9, n_estimators=100, max_depth=5, learning_rate=0.1`
- Threshold tuned to **0.764** via precision-recall curve (F1: 0.9001 → 0.9069)
- Generalization gap: -0.0019 AUC — no meaningful overfitting

### NB04 — Fairness Analysis

**Goals:** Measure disparate impact, apply Chouldechova impossibility, mitigate, summarize residual risks.

**Fairness metrics computed per group:**
- Demographic parity (approval rate, Adverse Impact Ratio)
- Equalized odds (TPR, FPR, FNR)
- Predictive parity (Precision, Brier score)
- AUC disaggregated

**Key findings:**
- Black applicants: AIR = 0.876 — fails EEOC 80% rule
- American Indian/Alaska Native: AIR = 0.859 — fails
- Native Hawaiian/Pacific Islander: AIR = 0.840 — fails
- FPR spread across racial groups: 0.21 (not equalized)
- Base rate spread: 38.4pp — Chouldechova impossibility confirmed

**Mitigation applied:** Group-specific threshold adjustment targeting FPR equalization at 0.365. FPR prioritized over FNR as most legally relevant under ECOA and the Fair Housing Act.

**Residual risks:**

| Risk | Severity | Mitigated? |
|---|---|---|
| Proxy discrimination | High | Partial |
| Sample selection bias | Medium | No |
| Intersectionality | Medium | No |
| Temporal drift | Medium | No |
| Chouldechova impossibility | Structural | By design |
| Feedback loop | High (if deployed) | No |
| Credit score model type as proxy | Low | Partial |
| GBT model opacity | Medium | Partial |

### NB05 — Explainability

**Goals:** Global and local explanations, proxy feature flags, counterfactual recourse.

**Methods:**
- **SHAP (TreeExplainer):** Global feature importance via Shapley values
- **LIME:** Local explanations for individual decisions
- **DiCE:** Counterfactual explanations for denied applicants

**Key findings:**
- SHAP top features: `debt_to_income_ratio` > `property_value` > `loan_purpose_32` > `loan_amount`
- No geographic proxy features (census_tract, county_code) appear in top 10 SHAP — model makes decisions on legitimate financial signals
- SHAP/LIME strong agreement: both rank DTI, property_value, and loan_purpose_32 in top 5
- DiCE counterfactuals: lowering DTI and changing loan purpose are the most common actionable changes for denied applicants

### NB06 — Robustness & Drift

**Goals:** Implement Lecture 04 robustness pipeline — PSI, KS, MMD, stress testing, ICE curves.

**Distribution drift (Part A):**
- PSI: All features STABLE (PSI = 0.00) — stratified split explains this
- KS test: no features marginally significant
- MMD² ≈ 0 — no meaningful distributional shift
- **Diagnosis: No significant drift detected**

**Generalization gap (Part B):**
- G_gap AUC = -0.0019 — excellent generalization, no overfitting
- No geographic proxy features flagged in permutation importance

**Stress testing — DFAST analog (Part C):**

| Scenario | Approval Rate Δ | AUC Δ |
|---|---|---|
| Baseline | — | 0.8515 |
| Income shock -20% | -1.0pp | -0.002 |
| Property value -20% | -1.0pp | -0.003 |
| **DTI +10 points** | **-10.3pp** | **-0.064** |
| **Combined adverse** | **-11.8pp** | **-0.068** |

**Key finding:** Model is highly sensitive to DTI increases — consistent with DTI being the top SHAP feature and concerning because DTI shocks disproportionately affect lower-income borrowers.

**Slice-based evaluation:**
- Black applicants: FNR = 0.225 vs. White: FNR = 0.188 — disparity persists post-model
- AUC consistent across groups (0.835–0.871)

## Is Deployment Defensible?

| Axis | Assessment |
|---|---|
| **Legal** | 80% rule fails for Black, AI/AN, and Native Hawaiian applicants. Group-specific mitigation improves but does not fully resolve. ECOA adverse action notices supportable via SHAP/LIME. |
| **Technical** | G_gap = -0.0019 (excellent generalization). No drift detected. Fully reproducible pipeline. Code versioned on GitHub. |
| **Ethical** | Top features are legitimate financial signals. No geographic proxies in top 10 SHAP. Counterfactuals provide actionable recourse. Feedback loop risk unresolved. |

**Verdict:** Conditionally defensible with mandatory monitoring:
- Monthly PSI on income and DTI
- Quarterly 80% rule audit by race/ethnicity
- Annual retraining review

### Memory Notes

The 2024 national HMDA LAR file is ~12.2M rows. The pipeline uses:
- **Chunked CSV loading** (50k rows at a time) to avoid OOM errors during load
- **500k stratified subsample** for model training and notebook 06 analysis
- `n_jobs=1` throughout to prevent joblib spawning memory-hungry parallel workers

If you encounter memory errors, reduce `CHUNK_SIZE` in NB01 to 25,000 or reduce `MAX_TRAIN` in NB03 to 200,000.

## Key Design Decisions

| Decision | Rationale |
|---|---|
| Gradient Boosting over Random Forest and Logistic Regression | Highest AUC (0.8515) and overall F1 (0.9001); tradeoff explicitly documented |
| Threshold 0.764 over default 0.5 | Precision-recall optimization; F1 improved from 0.9001 to 0.9069 |
| FPR equalization over FNR or calibration | Most legally relevant under ECOA; prevents wrongful denials |
| 500k subsample for training | Computational tractability; stratified to preserve class balance |
| Protected attributes excluded from X | Core responsible ML principle; preserved separately for post-hoc auditing |
| Median imputation on train only | Prevents leakage; medians saved for production reproducibility |
| `interest_rate` removed from features | 98.4% missing for denied apps vs. 0.3% for approved — median imputation encodes outcome directly |
| Credit score type collapsed to binary | 15+ dummy columns from multi-category encoding; binary preserves signal without bloat |
